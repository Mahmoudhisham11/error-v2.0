# نظام إدارة الخطوط والعمليات المالية

## 📋 نظرة عامة على المشروع

هذا المشروع هو **تطبيق ويب متقدم (PWA)** مبني باستخدام **Next.js 16** لإدارة الخطوط الهاتفية والعمليات المالية (الإرسال والاستلام). التطبيق يستخدم **Firebase Firestore** كقاعدة بيانات ويوفر نظام إدارة كامل للعمليات اليومية والتقارير والمصاريف مع واجهة مستخدم عربية كاملة.

### فكرة المشروع

المشروع عبارة عن نظام إدارة مالي متكامل لإدارة:
- **الخطوط الهاتفية**: إدارة الخطوط مع حدود شهرية للإرسال والاستلام
- **العمليات المالية**: تسجيل عمليات الإرسال والاستلام مع حساب العمولات
- **التقارير**: عرض تقارير يومية مفصلة للأرباح والعمليات
- **المصاريف**: إدارة المصاريف اليومية والتاريخية
- **لوحة التحكم**: عرض إحصائيات شاملة عن رأس المال والأرباح والمصاريف
- **النقدي**: إدارة رصيد النقدي والتحقق منه قبل تنفيذ العمليات

## 🎯 الصفحات والمكونات الرئيسية

### 1. الصفحة الرئيسية (`app/page.jsx`)

**الوظيفة:** نقطة الدخول الرئيسية للتطبيق

**الميزات:**
- التحقق من حالة تسجيل الدخول
- عرض صفحة تسجيل الدخول أو إعادة التوجيه للعمليات
- استخدام `localStorage` للتحقق من بيانات المستخدم

---

### 2. صفحة تسجيل الدخول (`components/Login/page.jsx`)

**الوظيفة:** إدارة تسجيل الدخول وإنشاء الحسابات

**الميزات:**

#### تسجيل الدخول:
- التحقق من اسم المستخدم وكلمة المرور واسم الفرع
- التحقق من وجود المستخدم في Firebase
- التحقق من صحة كلمة المرور
- التحقق من حالة تفعيل الحساب (`permissions.active`)
- حفظ بيانات المستخدم في `localStorage`

#### إنشاء حساب جديد:
- التحقق من عدم وجود اسم المستخدم مسبقاً
- إنشاء حساب مع الصلاحيات الافتراضية
- الصلاحيات: `{cards: false, reports: false, active: false, sittings: false}`

---

### 3. صفحة العمليات (`app/operations/page.jsx`)

**الوظيفة:** إدارة العمليات اليومية (الإرسال والاستلام)

**الميزات:**

#### عرض البيانات:
- عرض جميع الخطوط المتاحة للفرع
- عرض جميع العمليات اليومية
- **Stat Cards**: عرض الربح، إجمالي المصاريف، صافي الربح

#### تنفيذ العمليات:
- **عملية الإرسال**:
  - خصم المبلغ من رصيد الخط
  - خصم المبلغ من حد الإرسال الشهري
  - إضافة المبلغ إلى رصيد النقدي
  - إضافة العملية إلى قاعدة البيانات

- **عملية الاستلام**:
  - إضافة المبلغ إلى رصيد الخط
  - خصم المبلغ من حد الاستلام الشهري
  - خصم المبلغ من رصيد النقدي
  - **التحقق من رصيد النقدي قبل التنفيذ**

#### التحقق من الحدود:
- التحقق من وجود رصيد كافي في المحفظة للإرسال
- التحقق من عدم تجاوز حد الإرسال الشهري
- التحقق من عدم تجاوز حد الاستلام الشهري
- **التحقق من رصيد النقدي قبل عملية الاستلام**

#### إدارة العمليات:
- حذف عملية مع إعادة الرصيد
- **تقفيل اليوم**: نقل العمليات والمصاريف إلى التقارير

---

### 4. صفحة الخطوط (`app/cards/page.jsx`)

**الوظيفة:** إدارة الخطوط الهاتفية

**الميزات:**

#### عرض الخطوط:
- عرض جميع الخطوط في جدول
- معلومات كل خط: اسم المالك، رقم الخط، الحدود، الرصيد، الرقم القومي
- حساب إجمالي الرصيد

#### البحث:
- البحث عن خط برقم الخط
- استخدام `datalist` لتسهيل البحث

#### إدارة الخطوط (للمستخدمين المصرح لهم):
- **إضافة خط جديد**: مع جميع البيانات والحدود
- **تعديل خط**: تحديث البيانات
- **حذف خط**: حذف من قاعدة البيانات

#### إدارة الحدود الشهرية:
- إعادة ضبط الحدود تلقائياً في بداية كل شهر
- الحفاظ على الحدود الأصلية
- حساب حد الاستلام بناءً على الرصيد

---

### 5. لوحة التحكم (`app/dashboard/page.jsx`)

**الوظيفة:** عرض إحصائيات شاملة وإدارة المصاريف التاريخية

**الميزات:**

#### Stat Cards (الصف الأول):
- **رأس المال**: إجمالي رأس المال
- **إجمالي النقدي**: رصيد النقدي الحالي
- **إجمالي رصيد المحافظ**: مجموع أرصدة جميع الخطوط

#### Stat Cards (الصف الثاني):
- **الربح**: إجمالي الربح من التقارير
- **إجمالي المصاريف**: إجمالي المصاريف التاريخية
- **صافي الربح**: الربح - المصاريف

#### المصاريف التاريخية:
- **Toggle Button**: إظهار/إخفاء المصاريف
- **بحث بالتاريخ**: تصفية المصاريف حسب التاريخ
- **جدول المصاريف**: عرض جميع المصاريف مع التفاصيل
- **زر حذف المصاريف**: حذف جميع المصاريف التاريخية

#### الخطوط التي تحتاج انتباه:
- عرض الخطوط التي تقترب من حدودها
- تحذيرات عند اقتراب الخط من الحد الأقصى

## 💾 هيكل قاعدة البيانات (Firebase Firestore)

### Collections المستخدمة:

#### 1. `users`
```javascript
{
  userName: string,
  password: string,        // نص عادي (يُنصح بتشفيره)
  shop: string,
  permissions: {
    cards: boolean,
    reports: boolean,
    sittings: boolean,
    active: boolean
  }
}
```

#### 2. `cards`
```javascript
{
  userName: string,
  phone: string,
  nationalId: string,
  amount: number,              // الرصيد الحالي
  depositLimit: number,        // حد الإرسال المتبقي
  withdrawLimit: number,       // حد الاستلام المتبقي
  originalDepositLimit: number, // الحد الأصلي للإرسال
  originalWithdrawLimit: number, // الحد الأصلي للاستلام
  shop: string,
  lastReset: Date             // آخر إعادة ضبط للحدود
}
```

#### 3. `operations`
```javascript
{
  source: 'line' | 'machine',  // نوع العملية (خط أو ماكينة)
  phone: string,                // رقم الخط أو كود الماكينة
  name: string,                 // اسم العميل أو المندوب
  amount: number,
  commation: number,            // العمولة
  shop: string,
  type: 'ارسال' | 'استلام' | 'deposit' | 'withdraw' | 'transfer' | 'topup' | 'bill',
  date: string,                 // ISO string
  dateString: string,           // تاريخ العملية (YYYY-MM-DD) للاستعلام السريع
  amountBefore: number,         // الرصيد قبل العملية
  closed: boolean,              // حالة التقفيل
  
  // حقول إضافية لعمليات المكينات فقط:
  machineCode?: string,         // كود الماكينة
  machineName?: string,         // اسم الماكينة
  balanceAfter?: number         // رصيد الماكينة بعد العملية
}
```

#### 4. `reports`
```javascript
{
  // نفس هيكل operations +:
  date: string                 // تاريخ التقفيل
}
```

#### 5. `expenses`
```javascript
{
  amount: number,
  reason: string,
  shop: string,
  date: string                 // ISO string
}
```

#### 6. `expensesReports`
```javascript
{
  amount: number,
  reason: string,
  shop: string,
  date: string,                // تاريخ التقفيل
  closedAt: string             // ISO string - وقت التقفيل
}
```

#### 7. `cash`
```javascript
{
  shop: string,
  amount: number,
  createdAt: Date,
  updatedAt: Date
}
```

## 🔄 نظام الحدود الشهرية

### آلية العمل:
1. عند إضافة خط جديد، يتم حفظ الحدود الأصلية
2. في بداية كل شهر، يتم إعادة ضبط الحدود تلقائياً
3. حد الاستلام يتم حسابه بناءً على الرصيد:
   ```
   withdrawLimit = originalWithdrawLimit - amount
   ```

### إعادة الضبط الشهرية:
- تحدث تلقائياً عند فتح صفحة الخطوط
- تحافظ على جميع العمليات السابقة
- تعيد الحدود إلى قيمها الأصلية
